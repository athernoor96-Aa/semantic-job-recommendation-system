1. Problem Statement¶

On a job website, people search using their own words. Plain keyword search misses good matches — for example "ML engineer" and "Data Scientist" are similar jobs but use different words. This frustrates candidates and recruiters.

Better matches mean more applications, happier candidates, and faster hiring for the company.

In a real company this would be the website's own job postings and user searches. Here I use a public stand-in: the 1.3M LinkedIn Jobs & Skills dataset, which has job titles, descriptions, and skills.

As an NLP task, I am do two things:

  1. Text classification (supervised): Predict a job's category from its description — this is the trained ML model.
  2. Semantic search (ranking): Turn text into vectors and rank jobs by similarity to a search query and returns the most similar jobs.

2. High-Level System Design¶

The pipeline is built as simple connected steps. Each step has one job and passes its result to the next:

What each step does and why I need it:

  1. Load the data — read the job postings from the CSV files into one table.
  2. Clean the text — make everything lowercase and remove links, numbers, and symbols so only useful words remain.
  3. TF-IDF — turn the words into numbers, giving each word a score based on how important it is.
  4. Train the model — teach a Logistic Regression model to predict a job's category from its text (the machine-learning part).
  5. Recommend jobs — for a search query, use cosine similarity to find the jobs with the most similar text.
  6. Evaluate — measure how well the model works using accuracy and F1-score

def to_category(title):
    t = str(title).lower()
    if any(w in t for w in ["data scientist", "machine learning", "data analyst", "analytics"]): return "Data & Analytics"
    if any(w in t for w in ["software", "developer", "engineer", "programmer"]):                 return "Software"
    if any(w in t for w in ["nurse", "clinical", "medical", "health", "care"]):                   return "Healthcare"
    if any(w in t for w in ["sales", "account manager", "business development"]):                  return "Sales"
    if any(w in t for w in ["account", "finance", "audit", "tax"]):                               return "Finance"
    if any(w in t for w in ["marketing", "seo", "content", "brand"]):                             return "Marketing"
    return "Other"

def clean_text(t):
    t = str(t).lower()
    t = re.sub(r"http\S+|<.*?>", " ", t)
    t = re.sub(r"[^a-z\s]", " ", t)
    t = re.sub(r"\s+", " ", t).strip()
    return t

df["text"] = (df["job_title"] + " " + df["job_summary"] + " " + df["job_skills"]).apply(clean_text)

7. TF-IDF: Turn Text into Numbers¶

TF-IDF gives each word a score (high score = important word). This single TF-IDF matrix is used by both the classifier and the recommender.

In [13]:
tfidf = TfidfVectorizer(max_features=5000, stop_words="english")

print("TF-IDF matrix shape (jobs x words):", X.shape)

TF-IDF matrix shape (jobs x words): (16740, 5000)

TF-IDF was picked because it is fast, intuitive, and efficient for text with many keywords. The reason for Logistic Regression being chosen as a classifier is that it is easy to understand, trains quickly from sparse text data and is easy to understand.

For the recommendation system, LSA, which is actually Truncated SVD, takes the sparse TF-IDF data and turns it into 100 dense components representing meaning, which enables jobs having similar wording to cluster together. This makes cosine similarity a good choice because it looks at the vector directions.

10. Semantic Meaning Vectors (LSA)¶

The semantic part, TF-IDF only matches exact words. To capture meaning, then I shrink the big TF-IDF matrix into 100 small numbers per job using TruncatedSVD (LSA). Jobs about similar things end up with similar vectors, even if they use different words. Normalise the vectors so I can compare them with cosine similarity.

11. Recommend Similar Jobs¶

Component 4. We turn the search text into a meaning vector and use cosine similarity to find the closest jobs.

In [17]:
def recommend(query, k=5):
    q = normalize(svd.transform(tfidf.transform([clean_text(query)])))
    sims = cosine_similarity(q, emb)[0]
    top = np.argsort(-sims)[:k]
    out = df.iloc[top][["job_title", "category"]].copy()
    out["score"] = sims[top].round(3)
    return out

recommend("python machine learning and data analysis", k=5)

12. Evaluate the Recommender (Ranking Metrics)¶

Examples look nice, but I should measure quality Then I pick some jobs as test "searches" and say a recommended job is correct (relevant) if it has the same category. Also, compute two easy, standard ranking scores:

  1. Precision@k — of the top k results, how many are the right category?
  2. MAP@k — like precision, but gives more credit when correct jobs are near the top
In [18]:
svd = TruncatedSVD(n_components=100, random_state=42)
emb = svd.fit_transform(X)

# safe normalize (avoids divide-by-zero -> no warnings)
norms = np.linalg.norm(emb, axis=1, keepdims=True)
norms[norms == 0] = 1
emb = emb / norms

print("Meaning-vector shape:", emb.shape)

In [19]:
def evaluate_recommender(n_queries=200, k=5, seed=42):
    rng = np.random.default_rng(seed)
    cats = df["category"].values
    precisions, average_precisions = [], []
    sample = rng.choice(len(df), size=min(n_queries, len(df)), replace=False)
    for qi in sample:
        sims = cosine_similarity(emb[qi:qi+1], emb)[0]
        order = np.argsort(-sims)
        order = order[order != qi][:k] 
        hits = [1 if cats[j] == cats[qi] else 0 for j in order]
        precisions.append(np.mean(hits))
        got, ap = 0, 0.0
        for i, h in enumerate(hits):
            if h:
                got += 1
                ap += got / (i + 1)
        average_precisions.append(ap / got if got else 0.0)
    print(f"Precision@{k}: {np.mean(precisions):.4f}")
    print(f"MAP@{k}:       {np.mean(average_precisions):.4f}")
evaluate_recommender()

13. Summary & Conclusion¶

What I built¶

A complete NLP pipeline with very simple code: preprocessing, clean text, TF-IDF, a trained Logistic Regression classifier (predicts job category), LSA meaning-vectors, a cosine-similarity recommender, evaluation of both parts.

How well it works¶

The classifier is checked with accuracy and F1, and the recommender with Precision@k and MAP@k — so we measure both components, not just show examples.

Strengths¶

  1. Covers the full brief (problem, design, components, model, evaluation, discussion).
  2. LSA adds real semantic matching, but the code stays short and easy.

Limitations¶

  1. LSA still works from words, so it can miss deep meaning and word order.
  2. The category label comes from title keywords, which is a simple approximation.
  3. Use sample of 50,000 rows for speed rather than using all 1.3M.

Future work¶

  1. Use Sentence-Transformer embeddings (e.g. all-MiniLM-L6-v2) for stronger meaning.
  2. Add more categories and use real user clicks to measure recommendation quality.
  3. Use a fast vector index (FAISS) to scale to the full dataset.
